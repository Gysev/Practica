# Практика 2.1 — Win32-приложение с иконкой в системном лотке

Задание: трей при запуске, ЛКМ — показать окно, ПКМ — меню «Открыть» / «Выход», обработка пересоздания панели задач (`TaskbarCreated`), запуск без главного окна (`--background` / `-hidden`), закрытие окна скрывает в трей (процесс продолжается), **Файл → Выход** завершает приложение, **один экземпляр** (named mutex), сборка **CMake** + MSVC, выходной **один exe** со статической CRT `/MT`.

> Доп. баллы WinUI 3 здесь не реализованы (реализовано на чистом Win32 — полностью покрывает требования методички без WinRT). При необходимости можно заменить оболочку на WinUI, сохранив логику трея через `TrayIcon`/WinAPI.

## Сборка (Windows, Visual Studio 2022)

Из корня репозитория (`Labs/`):

```powershell
cd practices/Practice2_1
cmake -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Release
```

Готовый бинарник: `build\Release\practice_tray_app.exe`.

### Альтернатива Ninja

Так же из `practices/Practice2_1/`:

```powershell
cmake -B build-ninja -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build-ninja
```

## Запуск

- Обычный: дважды клик по `practice_tray_app.exe` — окно видно сразу, иконка в трее есть.
- **Без окна при старте:**  
  `practice_tray_app.exe --background` (или `-hidden`)

## Проверка требования «пересоздание панели задач»

Перезапустите процесс проводника (например, Диспетчер задач → Windows Explorer → Перезапустить): иконка в трее должна появиться снова после рассылки `TaskbarCreated`.

## Одновременный запуск второй копии

Второй экземпляр показывает сообщение и **сразу выходит** до регистрации иконки.
