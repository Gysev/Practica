## ER-модель модуля лицензий (PostgreSQL)

Краткая схема для описания ER-диаграммы и миграций.

### Сущности

- **users** — уже есть в приложении (`AppUser`), владелец лицензии.
- **devices** — устройство пользователя (`Device`).
  - `id` bigint PK  
  - `external_device_id` varchar(256) UNIQUE в связке с владельцем (в коде фильтруется через `owner_user_id`)  
  - `owner_user_id` bigint FK → `users(id)`

- **software_licenses** — ключ продукта и срок действия (`SoftwareLicense`).
  - `id` bigint PK  
  - `license_key` varchar(64) UNIQUE NOT NULL — публичный UUID ключа лицензии  
  - `user_id` bigint FK NOT NULL → `users`  
  - `device_id` bigint FK nullable → `devices` (заполняется при активации)  
  - `status` varchar — `CREATED | ACTIVE | EXPIRED | REVOKED`  
  - `created_at` timestamp  
  - `activated_at` timestamp nullable  
  - `valid_until` timestamp nullable  
  - `validity_period_days` int NOT NULL — длительность, задаётся при создании или увеличивается при продлении в состоянии CREATED  
  - `blocked` boolean NOT NULL

### Связи

- Один пользователь — много устройств (`devices.owner_user_id` → `users`).
- Одна лицензия — один владелец (`software_licenses.user_id`).
- При активации лицензия ссылается ровно на одно устройство (`device_id`), по бизнес-правилу «одна активная привязка за раз» (упрощённая модель для практики).

### DDL (PostgreSQL)

```sql
CREATE TABLE devices (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  external_device_id VARCHAR(256) NOT NULL,
  owner_user_id BIGINT NOT NULL REFERENCES users(id)
);

CREATE UNIQUE INDEX ux_devices_owner_external ON devices (owner_user_id, external_device_id);

CREATE TABLE software_licenses (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  license_key VARCHAR(64) NOT NULL UNIQUE,
  user_id BIGINT NOT NULL REFERENCES users(id),
  device_id BIGINT REFERENCES devices(id),
  status VARCHAR(32) NOT NULL,
  created_at TIMESTAMP,
  activated_at TIMESTAMP,
  validity_period_days INTEGER NOT NULL,
  valid_until TIMESTAMP,
  blocked BOOLEAN NOT NULL DEFAULT FALSE
);
```

Hibernate создаёт аналогичные таблицы при `ddl-auto: update`; для боевого стенда лучше оформить отдельные миграции под вашу финальную ER.
