## ER-модуль антивирусных сигнатур (PostgreSQL)

Схема для отчёта и целевой миграции под профиль `postgres`.

### Сущности

- **antivirus_signatures** (`AntivirusSignature`): правило сигнатуры, логический флаг удаления, версия содержимого, ЭЦП по канону JCS над полями подписания.
  - `id` bigint PK  
  - `name` varchar(256) UNIQUE NOT NULL  
  - `content` text/clob NOT NULL  
  - `version` int NOT NULL (инкремент при успешном обновлении)  
  - `eds_signature` varchar(1024) NOT NULL — Base64 ЭЦП (RSA SHA-256 PKCS#1)  
  - `deleted` boolean NOT NULL DEFAULT FALSE  
  - `deleted_at` timestamp with time zone NULL  
  - `created_at`, `updated_at` timestamp with time zone NOT NULL (индекс по `updated_at` для инкрементальной выгрузки)

- **antivirus_signature_history**: снимки до/после для изменений (**CREATE**, **UPDATE**, **DELETE**); для создания поле «до» NULL.
  - `id` bigint PK  
  - `signature_id` bigint NOT NULL FK → `antivirus_signatures(id)` (идентификатор остаётся тем же после soft delete)  
  - `change_type` varchar(32) NOT NULL — `CREATE | UPDATE | DELETE`  
  - `before_json`, `after_json` text/clob  
  - `created_at` timestamptz NOT NULL  
  - `performed_by` varchar(128) NOT NULL

- **antivirus_signature_audit**: журнал операций (**CREATE**, **UPDATE**, **DELETE**) с кратким текстом действия.
  - `id` bigint PK  
  - `signature_id` bigint NOT NULL FK → `antivirus_signatures(id)`  
  - `action` varchar(32) NOT NULL — `CREATE | UPDATE | DELETE`  
  - `details` varchar(1024) NOT NULL  
  - `created_at` timestamptz NOT NULL  
  - `performed_by` varchar(128) NOT NULL

Индексы: по `signature_id` в истории и аудите (`idx_av_hist_sig`, `idx_av_audit_sig`), по `updated_at` у основной таблицы.

### Правила предметной области (задание 1.4)

- **Создание и обновление** — перерасчёт ЭЦП (канонический JSON с полями `content`, `name`, `signatureId`, `version` в ASCII-порядке ключей через RFC 8785 JCS → `SHA256withRSA`).
- **Удаление** — только логическое (`deleted = true`, `deleted_at`), ЭЦП **не пересчитывается**.
- При **CREATE** запись в History и Audit; при **UPDATE/DELETE** — History + Audit; каждая операция мутации пишет хотя бы одну строку в Audit.
- **Полная выгрузка** — только строки с `deleted = false`.
- **Инкремент** — строки с `updated_at > :since`, **включая** удалённые.
- По `signature_id` можно получить упорядоченные списки History и Audit (если строки в главной таблице нет, эндпоинт вернёт 404).

### DDL (PostgreSQL, ориентир)

Типы `ENUM` ниже можно заменить на `VARCHAR(32)`, как делает Hibernate в dev.

```sql
CREATE TABLE antivirus_signatures (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name VARCHAR(256) NOT NULL UNIQUE,
  content TEXT NOT NULL,
  version INT NOT NULL,
  eds_signature VARCHAR(1024) NOT NULL,
  deleted BOOLEAN NOT NULL DEFAULT FALSE,
  deleted_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE NOT NULL
);

CREATE INDEX idx_antivirus_signatures_updated_at ON antivirus_signatures (updated_at);

CREATE TYPE signature_change_kind AS ENUM ('CREATE', 'UPDATE', 'DELETE');

CREATE TABLE antivirus_signature_history (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  signature_id BIGINT NOT NULL REFERENCES antivirus_signatures(id),
  change_type signature_change_kind NOT NULL,
  before_json TEXT,
  after_json TEXT,
  created_at TIMESTAMP WITH TIME ZONE NOT NULL,
  performed_by VARCHAR(128) NOT NULL
);

CREATE INDEX idx_av_hist_sig ON antivirus_signature_history (signature_id);

CREATE TABLE antivirus_signature_audit (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  signature_id BIGINT NOT NULL REFERENCES antivirus_signatures(id),
  action signature_change_kind NOT NULL,
  details VARCHAR(1024) NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE NOT NULL,
  performed_by VARCHAR(128) NOT NULL
);

CREATE INDEX idx_av_audit_sig ON antivirus_signature_audit (signature_id);
```

Hibernate создаёт аналог при `ddl-auto: update`; для продакена предпочтительны управляемые миграции (Flyway/Liquibase).
